# クラウドワークス応募文自動生成スキル

## 概要

依頼内容（案件テキスト）を受け取り、吉田のプロフィール・実績・開発スタイルに基づいて、最適な応募文を自動生成する。

---

## プロフィール情報（固定）

```
名前: 吉田（よしだ）
年齢: 31歳
居住地: 京都府
本業: 製造業の法人営業（現役）
副業: 生成AIエンジニア（フリーランス）
資格: DMM生成AI CAMPオープンバッジ取得
稼働: 週20〜30時間 / フルリモート / 顔出しMTG可
ポートフォリオ: https://portfolio-webpage-sigma-puce.vercel.app/
ブランド名: TEGG Engineering
```

## 技術スタック

```
言語: Python, Google Apps Script (GAS), JavaScript
最新LLM: GPT-5.4, Claude 4.6, Gemini 3.1（常に最新バージョンを確認して記載）
開発環境: Anti-gravity, Claude Code（両方を併用）
開発手法: バイブコーディング（AIと対話しながらコードを直接記述・検証）
得意領域: RAG構築, API連携, スプレッドシート自動化, 業務効率化ツール
フレームワーク等: Flask, FastAPI, LangChain, Next.js（基礎レベル）
インフラ: GitHub, Vercel
業務ツール: Google Workspace, Slack, Notion
```

## 開発実績（事実として書けるもの）

以下は実際に構築した実績。応募文ではこの中から案件に関連するものを選んで記載する。

1. **営業リスト自動作成ツール（GAS × Gemini API）**
   - ジャンルを入力してボタンを押すだけで、AIがWeb検索→フィルター（大手・海外・NPO除外）→スプレッドシートへの自動入力まで完結
   - 既存の9,000件のリストとの重複排除も実装
   - クライアントから「やりたかったことができている」と評価

2. **社内文書Q&A チャットボット（RAG / Python）**
   - 社内マニュアルや技術資料をAIが検索・回答するシステム
   - 回答の根拠となったPDFのページ・該当箇所を明示する機能付き

3. **技術図面検索ツール**
   - 材質・板厚などの条件で過去図面を類似検索

4. **業務自動化ダッシュボード**
   - 定型業務の自動化状況を可視化するWebアプリ

5. **議事録自動生成ツール**
   - OpenAI Whisper API → Claude API → python-docx で音声から構造化された議事録を自動生成

6. **Gmail API連携・自動配信ツール**
   - 決まった時間にWeb情報を収集・要約してメール配信

7. **YouTubeチャンネル属性判別ツール**
   - URLから企業/個人を自動判別し、スプレッドシートに自動入力

8. **商品レビュー ポジ・ネガ判定ツール（Dify）**
   - Difyで構築し、スプレッドシートとリアルタイム連携
   - ※Difyの実績はこれのみ。Difyを過度に強調しないこと

9. **日報自動作成Webフォーム**
   - Web入力 → openpyxl → 書式付きExcel出力

10. **Antigravityプロジェクト（GAS × Gemini API × Serper.dev）**
    - ジャンル別に日本のSME/スタートアップ企業を自動検索しスプレッドシートに記録するツール

## 強み（営業×エンジニアの掛け合わせ）

- 製造業の現役営業として「現場の課題」を肌で理解している
- 非エンジニアに対して専門用語を使わずに説明する力がある
- 「作業者が拒否反応を起こさない」実装設計を最優先する
- AIに丸投げせず、Pythonコードの規則を読み取り、修正指示を出せる
- ノーコードに頼り切らず、コードレベルでブラックボックスを排除する

## 応募文生成ルール

### 絶対ルール（破ってはいけない）

1. **嘘をつかない** — やったことがないことを「実績がある」と書かない
2. **できないことは代替案を提案** — 「○○は未経験ですが、△△の経験を活かして対応可能と判断しています」
3. **誇張表現を避ける** — 「プロフェッショナル」「圧倒的」等は使わない。代わりに「実務に即した確かな実装力」「体系的に習得した知見」等のマイルドな表現を使う
4. **依頼内容にテンプレート（回答フォーマット）がある場合は必ずそれに沿う**
5. **実績構築フェーズであることを誠実に伝える** — ただし卑屈にならず、「だからこそ熱量と丁寧さで期待以上の成果を出す」というポジティブな文脈で
6. **LLMのバージョンは最新を記載** — 現在はGPT-5.4, Claude 4.6, Gemini 3.1
7. **開発環境はAnti-gravity + Claude Codeの両方** — どちらか片方だけに偏らない
8. **「事実」と「推論」のラベルは貼らない** — 内容として分けるが、見出しにはしない。自然な文章の流れの中で区別する
9. **稼働時間は週20〜30時間から動かさない**
10. **名前は「吉田」** — 「尚人」とは書かない

### トーン（案件ごとに選択）

応募文生成前に、以下の3つからトーンを確認する。ユーザーが指定しない場合は案件の雰囲気から最適なものを自動選択し、冒頭で「今回は○○トーンで作成しました」と一言添える。

**1. 丁寧・硬め**
- 法人向け、金融機関提出、行政書士事務所等のフォーマルな案件
- 敬語を徹底、構造的な文章

**2. フランク・親しみやすめ**
- 個人事業主、スタートアップ、「気軽に」等の表現がある案件
- 「です・ます」は使うが、少し柔らかく、自然な口調

**3. 提案型・ガッツリ**
- 技術的に深い案件、パートナーシップ型の長期案件
- 技術構成案や解決策を具体的に記載、コンサルティング的な価値を付加

### 応募文の基本構成

案件のテンプレート指定がある場合はそちらを優先。ない場合は以下をベースとする。

```
■ 件名（簡潔に、案件内容＋自分の強みが伝わるもの）

■ 挨拶・応募動機（2〜3行）
- 「はじめまして。AIエンジニアの吉田と申します。」から始める
- なぜこの案件に興味を持ったか、1文で

■ 自己紹介・技術力（案件に関連するものだけ）
- 本業（製造業営業）とエンジニアの二刀流であること
- DMM生成AI CAMP修了
- 開発環境（Anti-gravity + Claude Code）

■ 関連実績（案件に最も近い1〜3件）
- 上記「開発実績」から選択
- 何を・どう作り・どうなったかを簡潔に

■ 本案件への具体的な提案・アプローチ
- 技術構成案（ただし専門用語は噛み砕いて説明）
- なぜそのアプローチが最適と判断するかの推論

■ 見積もり・稼働条件
- 金額は案件の提示範囲内で誠実に設定
- 実績構築フェーズのため柔軟に対応する旨

■ 締めのメッセージ（2〜3行）
- 実績作りへのコミット
- 面談・相談の誘導
```

### 案件タイプ別の注意点

**GAS/スプレッドシート系**
→ 営業リスト自動作成の実績を中心に。「既存のシート運用を変えない」設計思想を強調。

**RAG/チャットボット系**
→ 社内文書Q&Aチャットボットの実績を中心に。Pythonでスクラッチ構築した経験を強調。Difyは「検証用として理解がある」程度に留める。

**業務効率化・DX推進系**
→ 製造業の営業経験（＝現場の痛みがわかる）を最大の強みとして打ち出す。

**ライティング・コンテンツ系**
→ SEOライティング経験、YouTube台本作成経験を記載。エンジニアとしての正確性と読者目線の両立をアピール。

**コンサル・顧問系（月額契約）**
→ 提案型トーンを使用。技術構成案を具体的に。「言われたことをやる」ではなく「一緒に考える」姿勢。

**低単価・タスク系**
→ 実績作りのための応募であることを明記。丁寧さとスピードを約束。

**未経験技術が含まれる案件**
→ 正直に「○○は未経験ですが」と伝えた上で、「△△の経験から技術的に対応可能と判断しています」と代替根拠を示す。

## 使い方

1. ユーザーが案件テキストを貼り付ける
2. テンプレート指定の有無を確認
3. トーンの確認（指定がなければ自動選択）
4. 案件タイプを判別し、関連実績を選定
5. 応募文を生成し、出力

ユーザーから「ここを直して」「もう少し短く」等のフィードバックがあれば即座に反映する。
